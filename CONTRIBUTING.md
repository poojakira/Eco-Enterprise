# Contributing to EcoTrack Enterprise

We welcome contributions from the industrial engineering community. To ensure the highest code quality standards, please follow these guidelines.

## Code Quality Standards
- **Strict Type Hinting**: All new functions must include Python type hints.
- **Linting**: Ensure all code passes `black` and `flake8` checks (run via CI).
- **Testing**: New features must include `pytest` units in `backend/tests`.

## Workflow
1. Fork the Industrial Nexus.
2. Create a feature branch (`git checkout -b feat/industrial-evolution`).
3. Commit with prefix (e.g., `feat:`, `fix:`, `infra:`).
4. Submit a Pull Request for review by the Absolute Reality team.
